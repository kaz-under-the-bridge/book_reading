# 事案タイムライン

株式会社関通ランサムウェア被害（2024年9月〜10月）の時系列。

## 攻撃前

| 時期 | 事象 |
|------|------|
| 2024年7月 | ファーストアタック（調査会社レポートより） |
| 2024年8月 | サーバーへの侵入の足跡を確認。SSL VPN機器の脆弱性を突かれ、ID/パスワードが漏洩 |

## 第1週（9/12〜9/18）— 発覚と緊急対応

| 日付 | 主な出来事 |
|------|-----------|
| **9/12（木）** | 夕方、ネットワーク完全ダウン。サーバーがランサムウェアに暗号化されたことを確認。「貴社のデータは暗号化された」のメッセージ |
| **9/13（金）** | 朝7時に役員緊急ミーティング。緊急対策室設置。出荷全面停止を決断。500社以上に影響。警察・保険会社・セキュリティ会社との連携開始。身代金不払いを即決 |
| **9/15（土）** | 損害保険会社・弁護士・セキュリティ専門家と緊急会議。保険適用範囲の確認。4社のインシデント対応企業が参画 |
| **9/15（日）** | 定例ミーティング開始。アナログ出荷・テザリング対応を模索。一部取引先で手書き送り状対応 |
| **9/16（月）** | セキュリティ対策会社の対応力に疑問 → 適切な会社の選定開始。社員への残業手当支給・管理監督者への手当も決定。金融機関と20億円の枠を設定 |
| **9/17（火）** | 「この戦いに必ず勝利する」宣言。9/23までアナログ対応、9/24からクラウドトーマスPro稼働、10/1外販完全復旧の目標設定。第二報プレスリリース発表 |
| **9/18（水）** | 東証への情報開示。ダークウェブでの個人情報漏洩監視体制構築。データセンター内サーバーは全廃棄を宣言。「空き巣に入られた家は捨てる」 |

| **9/22（日）** | 「やったらアカンこと」全社通達。社内インフラ復旧計画発表（EDR全台導入、クラウド一元管理等）。七連勤問題への対応策決定 |

## 被害の全体像

- **サーバー**: 全暗号化（データ使用不能）
- **バックアップ**: 一部破損（保存期間も短く復旧困難）
- **影響取引先**: 500社以上
- **解約**: 数件（大半の取引先は継続）
- **損害額**: 数億円規模
- **復旧期間**: 約1ヶ月で主要業務再開、完全復旧はさらに後
- **侵入経路**: SSL VPN機器の脆弱性
