## 第5章 多様な環境で成功を収める

この本の**第5章「多様な環境で成功を収める」**では、ソフトウェアエンジニアがキャリアを積む上で遭遇する可能性のある**様々なチーム構造、ビジネス環境（「平時」と「戦時」）、および企業の種類**について解説しています。

この章は、エンジニアが自身の働く場所や環境を深く理解し、その文脈に合った成功戦略を立てるのに役立つよう構成されています。

第5章は以下の4つの主要なトピックで構成されています。

### 1. プロダクトチームとプロダクト志向のエンジニア

プロダクトチームは、**外部の顧客向けのプロダクト**を開発するエンジニアリングチームです。

*   プロダクトチームで最も大きなインパクトを生むのは、**「プロダクト志向のエンジニア」**と著者によって定義されるエンジニアたちです。
*   プロダクト志向のエンジニアは、技術的な専門知識だけでなく、プロダクトが**顧客にどのような価値を提供するのか**を理解することに関心があります。
*   プロダクトマネージャーにプロダクトに関する質問をしたり、時間をかけて関係を築いたり、**プロダクトに関する業務に関わりたい**という意思を明確に伝えることが推奨されています。
*   ユーザー研究調査、カスタマーサポート、関連業務に関与することで、プロダクトがどのように機能するのかを学ぶと良いとされています。
*   プロダクトチームで働くことは、多方面にわたり円熟したエンジニアになるために、自身の馴染みのない種類のチームで働くことを検討する一つの機会になります。

### 2. プラットフォームチーム

プラットフォームチームは、**社内（内部）の顧客**を対象とし、他のエンジニアリングチームがプロダクト開発に貢献するためのサービスを提供します。

*   プロダクトチームとプラットフォームチームの両方で働いた経験を持つことで、**両チームへの理解が深まり**、それぞれが直面する課題を深く理解できるようになります。
*   エンジニアは、プロダクトチームとプラットフォームチームのどちらで働きたいのかが分かってくることでしょう。

### 3. 「平時」対「戦時」

企業は、そのビジネス環境やプレッシャーの度合いによって「戦時」モードと「平時」モードで運営されることがあります。ベンチャーキャピタル企業アンドリーセン・ホロウィッツの共同創業者であるベン・ホロウィッツが提唱した概念です。

| 分野 | 戦時 (War Time) | 平時 (Peace Time) |
| :--- | :--- | :--- |
| **ビジネス環境** | 存続へのプレッシャーがある。 | ほぼプレッシャーがない。 |
| **最優先事項** | **迅速にリリースすること**。通常、速度は品質よりはるかに重要であり、「完了は完璧に優る」という原則に従う。 | 機能を徹底的に検証した上でリリースすること。**品質の高い成果**を得ることを優先する。 |
| **会議** | 仕事をやり遂げるのを優先し、会議を減らす。 | 全員の意見を一致させるために会議を多数行うのが普通。 |
| **ワークライフバランス (WLB)** | ほとんど考慮されない（「ワーク・ライフ・何だって？」）。 | かなりWLBを重視し、WLBに配慮したポリシーを持つのが一般的。 |

*   「戦時」に成功を収めるためには、**仕事を迅速にやり遂げること**に集中し、対立を恐れずに（ただし個人的攻撃と受け止めずに）対処し、**燃え尽き**に注意を払うべきです。
*   「平時」に成功を収めるためには、**品質の高い成果**を優先し、人間関係の不満はプロフェッショナルでない行いと見なされないよう注意し、**仕事に飽きてしまう（停滞）**リスクの回避に努めるべきです。
*   ほとんどの企業は、ベンチャーキャピタルからの出資を受け、最終的にIPO（株式公開）に成功したスタートアップのように、**戦時モードと平時モードを交互に繰り返します**。例えば、創業時や資金調達ラウンドの間、IPOの1年前などは「戦時」となる傾向があります。

### 4. 企業の種類

ソフトウェアエンジニアリングのキャリアは、所属する企業の規模や種類によって大きく異なります。

*   **ビッグテックと大手テック企業:** 大半が「平時」モードで運営されている傾向があり、人々の出入りが多いため、**人脈**を築くことが将来の助けとなる可能性があります。
*   **ミドルからレイターステージのスケールアップ:** シリーズB、C、Dなどの資金調達段階にあり、まだ上場していない企業を指し、その圧倒的多数が**プロダクトチーム**を持っています。
*   **スタートアップ:** プロダクト・マーケット・フィットを目指して活動する企業で、本質的にリスクを伴いますが、労働環境は創業者の影響を大きく受けます。株式を提供するスタートアップは**ハイリスク／ハイリターン**な職場です。
*   **非テック企業:** テック部門を持つ伝統的な大企業は、ビッグテックやスケールアップよりも**予測可能なワークライフバランス**を提供する傾向があり、ソフトウェアエンジニアの在籍期間が長くなる場合があります。
*   **ベンチャーキャピタルの出資を受けていない小規模企業:** 投資家からの成長へのプレッシャーがなく、利益を上げなければ事業は失敗に終わるという特徴があります。仕事のペースがゆったりとしており、**安定した職場**である可能性があります。
*   **公共部門:** 安定性があり、報酬は明確に伝えられ、休暇や福利厚生が充実していることが多いです。
*   **コンサルティング企業、アウトソーシング派遣企業、受託開発者エージェンシー:** **採用されるのが最も容易**な職場で、特に経験が浅い場合に当てはまります。報酬は他の企業より低い場合が多いですが、多様なプロジェクトに携わる機会があります。ただし、シニアエンジニアからスタッフエンジニアへの昇進の道を提供していない場合があるという短所もあります。

この章で紹介された様々な環境を理解することは、エンジニアが自分のキャリアの立ち位置について考える際の、肩書きや報酬以外の重要な要素（仕事の満足度、人間関係、会社の文化、成長の機会、健康、柔軟性など）を検討する助けとなります。

***

**比喩的表現による補足：**

キャリアを切り開くことは、気象条件や船の構造が異なる様々な海域を航海するようなものです。穏やかな「平時」の海（安定した大企業）では品質と計画が重要ですが、荒れた「戦時」の海（スタートアップ）ではスピードと決断力が生命線になります。また、乗る船（企業の種類やチーム構造）によって、期待される役割（プロダクト志向かプラットフォーム志向か）が大きく変わるため、自分が今どの海にいて、どんな船に乗っているのかを理解することが、航海を成功させる鍵となります。
