# 3. 「平時」対「戦時」

## 概要
企業は、その存続へのプレッシャーや開発の最優先事項に応じて、「戦時」モードと「平時」モードで運営されることがあります。これらのモードによって、求められる行動、会議の頻度、ワークライフバランス（WLB）への配慮が大きく異なります。

## 詳細
| 分野 | 戦時 (War Time) | 平時 (Peace Time) |
| :--- | :--- | :--- |
| **ビジネス環境** | **存続へのプレッシャー**がある。競合他社に執着する。 | ほぼプレッシャーがない。競合他社に執着する必要なし。 |
| **開発チームの最優先事項** | **迅速にリリースすること**。「完了は完璧に優る」の原則に従う。 | 機能を徹底的に検証した上でリリースすること。**品質の高い成果**を優先する。 |
| **会議** | 仕事をやり遂げるのを優先し、会議を減らす。 | 全員の意見を一致させるために会議を多数行うのが普通。 |
| **ワークライフバランス (WLB)** | ほとんど考慮されない（「ワーク・ライフ・何だって？」）。 | かなりWLBを重視し、WLBに配慮したポリシーを持つのが一般的。 |
*   **戦時で成功するには：** **仕事を迅速にやり遂げること**に集中し、論争を恐れずに対処し（個人的攻撃と受け止めずに）、**燃え尽き**に注意を払うべきです。
*   **平時で成功するには：** **品質の高い成果**を優先し、人間関係の不満はプロフェッショナルでない行いと見なされないよう注意し、**仕事に飽きてしまう（停滞）**リスクの回避に努めるべきです。
*   ほとんどの企業は、ベンチャーキャピタルからの出資を受け、最終的にIPO（株式公開）に成功したスタートアップのように、**戦時モードと平時モードを交互に繰り返します**。例えば、創業時や資金調達ラウンドの間、IPOの1年前などは「戦時」となる傾向があります。

## ポイント
企業が現在どのモードにあるかを理解し、**「戦時」ならスピード**を、「**平時」なら品質と人間関係**を優先するなど、その状況に合わせた戦略を適用することが成功の鍵となります。
