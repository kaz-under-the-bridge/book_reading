# 第6章のまとめ
## 第6章 採用面接

提供された資料の抜粋には、**第6章「採用面接」**の具体的な詳細な内容（概要、詳細、ポイント）をまとめるための十分な情報が含まれていません。

ただし、**第6章**に関連すると思われる、採用プロセス全般および面接の種類に関する情報が、他の章や文脈から見つけられます。

### 採用プロセス全般と面接の種類に関連する情報

1.  **採用プロセスの構造：**
    中規模以上のテック企業は、似通った面接プロセスを用いる傾向があり、これはGoogleの面接方式に似ていることから「Google風」と呼ばれることもあります。

2.  **面接の段階と担当者：**
    面接プロセスは、通常以下の段階で構成されます。

    *   **一次選考**（レジュメ選考、一次電話面接）：リクルーター、インバウンドソーサー（応募書類の審査を専任で行う採用職）、ハイアリングマネージャーが担当します。
    *   **技術的選考**（電話面接による技術面での選考、宿題として出されるコーディング課題）：ソフトウェアエンジニア、ハイアリングマネージャーが担当します。
    *   **オンサイト面接**（最終段階）：
        *   コーディング面接（ソフトウェアエンジニアが担当）
        *   システム設計面接（アーキテクチャー面接とも呼ばれる）（ソフトウェアエンジニアが担当）
        *   ハイアリングマネージャーによる面接（品行面の面接とも呼ばれる）
        *   バーレイザーによる面接（専門のバーレイザー面接担当者、ハイアリングマネージャーが担当）

3.  **技術的選考（コーディング課題）:**
    技術的選考の目的は、候補者が職務に必要な基本的な資格を備えているか、コーディングを十分に実践しているか、そして最終面接に呼ぶ価値があるかを判断することです。ほとんどの場合、ソフトウェアエンジニアが担当します。

    *   **非同期的な課題への対処:** 非同期的に行われるコーディング課題にはかなりの時間をかける必要があるため、事前に計画を立て、リクルーターに自分の状況を伝えて時間軸に合わせてもらう交渉が必要です。
    *   **フィードバックの確認:** 時間を割く前に、不合格の場合に書面によるフィードバックが得られるかどうかをリクルーターに確認しておくことが推奨されます。

4.  **上級職（スタッフ＋）の面接:**
    スタッフレベル以上の面接では、以下のような面接が追加される傾向があります。
    *   特定分野に掘り下げる面接（専門知識の深さを確認）。
    *   プロダクトマネージャーによる面接（プロダクト担当者とうまく仕事ができそうかを評価）。
    *   ディレクターレベルの人物による面接。
    *   二回目のシステム設計面接。

5.  **バーレイザー:**
    昇進の決定方法に関連して、「次のレベルに期待される基準をどのように引き上げるのか」を問うことで昇進を正当化する組織もある、という言及があります（「バーレイザーを参照」）。バーレイザーは、オンサイト面接の担当者の一つとしてリストされています。

これらの情報は、第6章が扱うテーマである「採用面接」において、読者が理解を深めるのに役立つ内容であると考えられます。
