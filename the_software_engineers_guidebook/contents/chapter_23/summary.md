# 第23章のまとめ

## 概要

ス タッフ＋エンジニアの役割には、自分自身やチームだけでなく、組織全体のエンジニアリングの速度と品質を向上させることが期待されます。本章では、コーディングへの関わり方、開発プロセスやプラクティスの改善、効率化ツールの導入、そしてコンプライアンスとセキュリティへの対応について解説されています。

## 詳細

*   **相変わらず行うコーディング**:
    *   ス タッフ＋レベルでもコーディング時間を確保すべきですが、戦略的に行う必要があります。難易度が高いタスク、広範な影響を持つタスク（新しいフレームワークやテストの導入、CI/CD改善など）、プロジェクト初期の方向付けなどに注力します,。
    *   コーディングを通じて模範を示し（「完了」の基準を満たす高品質なコード）、ペアプログラミングを通じてメンタリングを行います,。
*   **有益なエンジニアリングプロセス**:
    *   「完了（Done）」の定義を明確にし、チーム内で合意形成を図ります。
    *   コーディングスタイルのガイドラインを設け、リンター等で自動化します。
    *   コードレビュー、自動テスト、スキャフォールディング（新規作成の自動化）、機能フラグの衛生管理（不要なフラグの削除）、システム健全性ダッシュボードの整備などを推進します,。
*   **迅速なイテレーションのためのプラクティス**:
    *   **CI/CD（継続的インテグレーション/デプロイ）**: フィードバックループを短縮し、自動テストやカナリアリリース、自動ロールバックを活用して信頼性を担保します,。
    *   **トランクベース開発**: 全員がメインブランチで作業し、機能フラグを使って未完成の機能を隠すことで、マージの複雑さを回避しリリース頻度を上げます,。
    *   **モノリポ vs マイクロサービス**: 大規模テック企業ではモノリポが一般的ですが、ツールへの投資が必要です。アーキテクチャは組織の規模やフェーズに合わせて選択します,。
*   **効率性を高めるツール**:
    *   **サービスカタログ**: サービスの所有者やオンコール担当を明確にします。
    *   **コード検索**: 大規模なコードベースを効率的に検索できるツール（Google Code Searchのようなもの）は生産性を大きく向上させます,。
    *   **開発者ポータル**: Backstageのように、ドキュメント、テンプレート、サービスカタログを一元化します。
    *   **クラウド開発環境（CDE）**: 環境構築の時間を短縮し、一貫性を保ちます。
    *   **AIコーディングツール**: Copilotなどのツールを活用し、コーディング、テスト作成、リファクタリングを効率化しますが、出力の検証は必須です,。
*   **コンプライアンスとセキュリティ**:
    *   GDPRなどの規制、PII（個人識別情報）の取り扱い、ログへの記録制限、監査対応など、ビジネスを守るためのルールを理解し遵守します,。
    *   セキュアなコーディング規約（OWASP等）に従い、依存関係の脆弱性管理やペネトレーションテスト（侵入テスト）に対応します,。

## ポイント

*   **戦略的な関与**: 自分の時間は限られているため、直接全てをコーディングするのではなく、難易度が高く影響範囲の広い部分に集中し、チーム全体の生産性を上げるためのツールやプロセス改善に投資します。
*   **「銀の弾丸」はない**: どのプラクティスやツール（マイクロサービスかモノリスか、自作か購入かなど）を採用するかは、組織の規模や課題に応じたトレードオフを考慮して決定する必要があります,。
*   **品質と速度の両立**: 自動化（テスト、CI/CD、ロールバック）への投資こそが、品質を落とさずに開発速度を上げるための鍵となります。
