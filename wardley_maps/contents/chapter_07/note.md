# 第7章 Note

## 全体所感

この章は著者の知的冒険の記録であると同時に、マッピングの理論的基盤が確立される決定的な章。拡散と進化の分離、Ubiquity vs Certaintyによる進化曲線の導出は、マッピングを「使える道具」から「理論に裏付けられたフレームワーク」へと格上げしている。著者の失業からの再起という個人的な物語も、キャリアの転機を経験した者には響く内容。

## 進化曲線とプラットフォームエンジニアリング

進化曲線（Genesis → Custom Built → Product → Commodity/Utility）は、プラットフォームエンジニアが日常的に扱うコンポーネントの分類にそのまま使える。

- **Genesis**: 社内で実験中の新技術（例: eBPFベースの観測ツール、AI Ops）
- **Custom Built**: 社内独自のCI/CDパイプライン、カスタムデプロイツール
- **Product**: Terraform、Kubernetes、GitHub Actions
- **Commodity/Utility**: AWS EC2、S3、CloudFlare

この分類ができると、各コンポーネントに対する投資判断が明確になる。「Custom Builtのままにしているが、市場にProductやCommodityが存在するコンポーネント」は、移行の優先候補。逆に、「まだGenesisだが将来的にProductになるコンポーネント」には先行投資の価値がある。

## 「拡散と進化は別物」という洞察

この章で最も重要な洞察。日本のIT業界でも、この混同は頻繁に見られる。

- 「クラウドはまだ早い」（2009年頃のコンサル）→ 拡散が初期段階であることと、進化段階が十分に進んでいることを混同
- 「AIは導入企業が少ない」→ 拡散が初期なのは正しいが、AI自体（特に特定の応用）はProduct段階に達している
- プラットフォームチームとして、経営層への説明で「他社の導入事例がない」という反論に対し、「拡散と進化は別のプロセス」と説明できるのは強力

## Purposeの進化と「コストセンターからの脱却」

著者のPurpose変遷は示唆に富む。

- Zimki時代: 「ユーティリティプラットフォームのプロバイダー」
- 研究時代: 「トポグラフィカルインテリジェンスをビジネスの世界に説明する」
- そして著者自身が「Purposeも進化する」と述べている

プラットフォームチームも同様にPurposeが進化すべき:
- 初期: 「インフラの安定運用」（Commodity/Utilityの提供）
- 中期: 「開発者生産性の向上」（Product段階のツール提供）
- 現在/未来: 「AIを活用したGTMエンジニアリングの実現」「開発者エクスペリエンスの最適化」

Purposeを固定化せず、ランドスケープの変化に応じて再定義し続けることが重要。

## ユーザーニーズの連鎖と情報の非対称性

Figure 69の「The chain」（自分のPurpose → ユーザー → ユーザーのユーザー）は、情報の非対称性の本質を視覚化している。

- プラットフォームチームのユーザーは開発チーム
- 開発チームのユーザーはエンドユーザー（顧客）
- プラットフォームチームがエンドユーザーのニーズを理解していないと、開発チームに対して的外れなサービスを提供することになる
- 逆に、経営層のニーズ（コスト最適化、コンプライアンス）を無視すると、著者のZimki失敗のように政治的に潰される

この「ニーズの連鎖」を意識することで、プラットフォームチームは多層的なステークホルダー管理ができるようになる。

## CIO/CTOの会話の軍事翻訳

著者がCIO/CTOの会話を軍事に翻訳するくだりは痛烈。

- 「クラウドは仮想化の一種」→「丘を爆撃するのは迫撃砲の一種」
- 「ビジネス誌によると成功企業がパイロットを検討中」→「将軍週刊誌によると成功した指揮官が今後のキャンペーンで検討中」

これは日本企業でも日常的に起きている。「他社の動向を見てから判断する」「コンサルの推奨を待つ」という姿勢は、軍事に翻訳すれば「他の軍隊がどう動くか見てから作戦を決める」に等しい。状況認識の欠如を補うために他者の真似をする -- これが戦略の不在。

## ポイント

進化曲線の理論的導出は、マッピングの信頼性を決定的に高める知見。しかし著者が繰り返し述べるように、進化曲線は「予見」であって「予測」ではない。何が起きるかは分かるが、いつ起きるかは分からない。この不確実性を受け入れつつ、進化の方向に備えるのが戦略の本質。プラットフォームチームとしては、コンポーネントの進化段階を定期的に再評価し、投資判断を反復的に更新し続けることが実践的な示唆。
