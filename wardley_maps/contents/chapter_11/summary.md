# 第11章 A Smorgasbord of the Slightly Useful（少し役に立つものの詰め合わせ）

> [Medium](https://medium.com/wardleymaps/a-smorgasbord-of-the-slightly-useful-2498a1163dd6) | PDF p.340-465

## 概要

本章はWardley Maps理論の総括であり、戦略を「マスター」するための体系的なフレームワークを提示する。ビジネスにおける「嘘」（効率化＝コスト削減、実行が戦略より重要、コアに集中すべき、等）を一つずつ論破した後、戦略の5ステップ（サイクルの理解→ランドスケープの学習→気候パターンの学習→ドクトリンの適用→ゲームプレイの適用）を詳述する。Wardley's Doctrineとして30近い普遍的原則を列挙し、さらにコンテキスト固有のゲームプレイ手法を紹介。生物学的レジリエンスの4類型（fragile/robust/fluid/resilient）も導入し、ビジネスを「生き物」として捉える視座を提供する。最終的に「すべてのモデルは間違いだが、有用なものもある」として読者をシナリオ演習に送り出す。

## セクション別サマリー

### 1. 戦略をマスターする5つのステップ

- **Step 1 — The cycle**: 戦略は連続サイクル。目的のwhy（なぜこのゲームに参加するか）と行動のwhy（なぜこの手を打つか）を区別する。OODAループや孫子の兵法と共通する（Figure 156参照 — Mediumで閲覧可能）
- **Step 2 — Learn the landscape**: マップを作成し環境を観察する。マップの必須要件は、visual・context specific・position relative to anchor・movement（Figure 157参照 — Mediumで閲覧可能）
- **Step 3 — Learn and use climatic patterns**: 気候パターンは全プレイヤーに影響する「ゲームのルール」。進化は供給と需要の競争で起こる、進化は時間では測定できない、単一の方法が全てに適合することはない、等
- **Step 4 — Learn and use doctrine**: 普遍的に適用すべき原則群（Wardley's Doctrine）。マップがドクトリンを解決してくれるもの、マップを使ってドクトリンを適用するもの、マップ上に直接適用するものの3種類（Figure 159参照 — Mediumで閲覧可能）
- **Step 5 — Learn and use gameplay**: コンテキスト固有の戦略手法。マップ上の位置に基づいて「どこを攻撃するか」を決定する（Figure 160参照 — Mediumで閲覧可能）

### 2. 気候パターンの全リスト

- すべては供給と需要の競争を通じて進化する
- 進化は多くのキャズムを持つ複数の拡散波で構成される
- 進化を時間や採用率で測定することはできない
- 進化が進んでいないほど不確実性が高い
- 進化について選択権はない（Red Queen効果）
- コモディティ化は中央集権化を意味しない
- コンポーネントの特性は進化に伴って変化する
- 単一の方法が全てに適合することはない
- コンポーネントは共進化できる
- 効率化はイノベーションを可能にする
- 高次システムは新しい価値の源泉を創出する
- 将来の価値は確実性と反比例する
- 効率化はコスト削減を意味しない（ジェボンズのパラドックス）
- 高次システムへの進化はエネルギー消費の増加をもたらす
- 資本は新しい価値の領域に流れる
- コミュニケーションの進化は全体の進化速度を加速する
- 変化は常に線形ではない
- プロダクトからユーティリティへの移行は断続平衡を示す
- 成功は慣性を生む / 慣性は成功するほど強くなる / 慣性は組織を殺す
- 創造的破壊
- 競合の行動はゲームを変える
- ほとんどの競合は状況認識が低い
- すべてがランダムではない
- 経済にはサイクルがある（peace-war-wonder）
- 2つの異なる形態のディスラプション
- 「戦争」（産業化のポイント）は組織を進化させる

### 3. Wardley's Doctrine（普遍的原則）

- **透明性**: マップを組織内で共有し、他者の知恵と挑戦を取り込む
- **共通言語**: 効果的なコラボレーションの前提条件
- **前提への挑戦**: エゴを捨て、より良いマップを目指す
- **ユーザーを知る / ユーザーニーズに焦点を合わせる**
- **FIRE原則**: Fast, Inexpensive, Restrained, Elegant
- **プラグマティック**: 既存のものを再発明しない
- **バイアスと重複の排除**: 組織内の重複は想像を超える規模
- **適切な方法とツールの使い分け**: Agile/Lean/Six Sigmaの使い分け
- **成果に焦点を合わせる**: 契約形態ではなく成果を重視
- **標準の活用**: 産業化されたコンポーネントには標準を使う
- **フローの最適化 / 有効性 > 効率性**
- **慣性の管理 / 失敗の管理**: Chaos Engineeringの活用
- **小さく考える**: 小さなチーム、小さな契約
- **権限の分散**: 判断を現場に委ねる
- **目的・熟達・自律性の提供**
- **適性と態度で考える**: Pioneer-Settler-Town Planner
- **単一の文化はない**: 複数の文化を共存させる
- **最高を追求する**: 優れた人材を適切な役割に配置
- **常時進化のための設計**: セルベースの組織、盗みのシステム
- **体系的な学習メカニズム**: マップを振り返りと学習に活用
- **エコシステムに耳を傾ける**: ILCモデル等でセンシング
- **行動へのバイアス**: 完璧なマップを目指さず、行動から学ぶ
- **新しいものへのバイアス / オーナーシップ / 反復的な戦略 / より少ないもので改善**
- **卓越した基準の設定 / 戦略は複雑 / 方向にコミットし、経路は柔軟に / 速く動く**
- **コアは存在しない**: 全ては一過性
- **景観を活用する / 大きく考える / 謙虚であれ**

### 4. ゲームプレイ（コンテキスト固有の戦略）

- Open approaches（オープン戦略）: 障壁を下げ、採用を加速
- IPR: 知的財産権で競争を制限し進化を遅らせる
- FUD（Fear, Uncertainty and Doubt）: 顧客の慣性を利用して変化を遅らせる
- Exploiting constraint: 制約を利用して競合を分断
- Sweat and Dump: レガシーを第三者に押し付ける
- Pig in a poke: レガシーを「将来のビジネス」として売却
- Two factor markets: プロバイダーとコンシューマーを結びつけ、ネットワーク効果を活用
- Sensing Engines (ILC): 産業化→エコシステム構築→消費データで次の候補を特定

### 5. 生物学から学ぶ — レジリエンスの4類型

- **Fragile（脆弱）**: 低いエンジニアリング・レジリエンス、低い生態学的レジリエンス
- **Robust（堅牢）**: 高いエンジニアリング・レジリエンス、低い生態学的レジリエンス
- **Fluid（流動的）**: 低いエンジニアリング・レジリエンス、高い生態学的レジリエンス
- **Resilient（回復力のある）**: 高いエンジニアリング・レジリエンス、高い生態学的レジリエンス
- 自然は進化と競争を通じてレジリエンスを獲得。多様性が生態学的レジリエンスの鍵

### 6. マップについて覚えておくこと

1. マップは常に変化する生きたドキュメント
2. マップは学習とコミュニケーションの手段。完璧さより継続的な学習が重要
3. すべてのモデルは間違いだが、有用なものもある
4. 戦略は単純なトピックではない。圧倒されても正常

## ポイント

この章は第8章の内容を発展させた「Wardley Maps理論の完全版リファレンス」である。気候パターン、ドクトリン、ゲームプレイの3層構造が戦略フレームワークの核心を構成する。特に重要なのは、ドクトリン（普遍的原則）とゲームプレイ（コンテキスト固有の戦略）の明確な区別。ドクトリンは全ての組織が適用すべきものだが、ゲームプレイはマップ上の位置に依存する。この区別を理解せずに「ベストプラクティス」を適用すると、状況に合わない戦略を選ぶリスクがある。
