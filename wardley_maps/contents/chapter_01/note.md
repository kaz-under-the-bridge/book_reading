# 第1章 Note

## 全体所感

著者の「戦略が分からないCEO」としての正直な告白は、CLAUDE.mdの「PdMのしごと」で書いた「情報の非対称性」の問題と完全に重なる。CEOですら戦略を理解していない（理解していると見せかけている）という構図は、日本企業のシニアマネジメントにも当てはまる。「よさそうに見える」で承認する群集心理は、稟議文化との親和性が高い。

## 5つの要素とプラットフォームエンジニアリング

孫子の5要素をPFEチームに当てはめると:

| 要素 | PFEへの読み替え |
|------|---------------|
| Purpose | 開発者の生産性向上・信頼性の確保 |
| Landscape | 技術スタック、チーム間依存関係、開発者のペインポイント |
| Climate | クラウドのコモディティ化、AI/LLMの台頭、規制変化 |
| Doctrine | CI/CD、IaC、SLO/SLI、ポストモーテム等の普遍的プラクティス |
| Leadership | 「今、何に投資すべきか」の文脈固有の判断 |

多くのPFEチームはDoctrineの整備（CI/CD構築、監視基盤等）に注力しがちだが、Landscapeの可視化なしにはその優先順位が正しいか判断できない。「Kubernetes導入」「サービスメッシュ導入」がDoctrineなのかLeadership判断なのかを区別できていない現場は多い。

## 「盤面なきチェス」の比喩

この比喩はプラットフォームチームの価値可視化の議論に直結する。経営層に「なぜこの基盤投資が必要か」を説明するとき、SWOT的な説明（「強み: 高い技術力、弱み: レガシーシステム」）では意思決定に使えない。代わりにマップで「このコンポーネントはコモディティ化しているからBuildではなくBuy」と示せれば、投資判断の根拠になる。

## 成果バイアスへの警戒

「Google/Netflixがやっているから正しい」は典型的な成果バイアス。彼らの文脈と自社の文脈が同じとは限らない。PdMの本で学んだ「ベストプラクティスのワーストなところ（第7章）」と同じメッセージ。

## ポイント

- 戦略の5要素フレームワークは、PFEチームの技術投資判断に直接適用できるポテンシャルがある
- 「盤面を見せる」ことこそがPFEチームのコストセンターからの脱却手段になりうる
- 次章以降でマップの具体的な描き方が出てくることを期待
